# QuizMark — Lightweight Quiz DSL

QuizMark is a handwritten parser and CLI for creating, validating, styling, and exporting quizzes in a simple `.qm` format. It is designed to be easy to embed in web pages and to export Moodle-compatible XML.

## Quick start

```bash
python -m quizmark validate examples/geography.qm
python -m quizmark export examples/geography.qm --format html --out quiz.html
python -m quizmark export examples/geography.qm --format web --out web_bundle
```

# Quiz Parser — ANTLR4 Project

A parser for a custom `.qm` file format, built with [ANTLR4](https://www.antlr.org/) and Python.

---

## How It Works — Pipeline Overview

```
quiz file (.qm)
      │
      ▼
  QuizLexer          tokenises raw text into a stream of tokens
      │
      ▼
  QuizParser         builds a parse tree according to Quiz.g4 grammar rules
      │
      ▼
  QuizVisitorImpl    walks the parse tree, returns typed Python dataclass objects
      │
      ▼
  print_quiz()       renders the result to the terminal
```

---

## File Structure

| File | Role |
|---|---|
| `Quiz.g4` | ANTLR grammar — defines the language |
| `QuizLexer.py` | **Generated** — breaks input text into tokens |
| `QuizParser.py` | **Generated** — parses token stream into a parse tree |
| `QuizListener.py` | **Generated** — base listener interface (enter/exit hooks per rule) |
| `QuizVisitor.py` | **Generated** — base visitor interface (one `visit*` method per rule) |
| `QuizVisitorImpl.py` | **Written** — concrete visitor, extracts data from the parse tree |
| `models.py` | **Written** — plain Python dataclasses representing the parsed data |
| `main.py` | **Written** — entry point, wires everything together |

The four `Quiz*.py` files are auto-generated by ANTLR and should **not** be edited by hand. Regenerate them with:

```bash
antlr4 -Dlanguage=Python3 -visitor Quiz.g4
```

## The Grammar (`Quiz.g4`)

The grammar describes what a valid `.qm` file looks like. It has two sections:

### Parser Rules (structure)

```
quiz          →  'QUIZ:' title themeBlock? metadata* question+
themeBlock    →  'THEME' '{' themeStatement+ '}'
themeStatement→  key '=' value
               | selector '[' attr=val, ... ']'
selector      →  'quiz' | 'question' | 'answer' | 'correct' | 'wrong'
metadata      →  IDENTIFIER ':' (text | NUMBER | BOOLEAN)
question      →  'QUESTION' points? ':' text image? answer+
points        →  '(' 'points=' NUMBER ')'
answer        →  OPTION_LABEL ':' answerContent '*'?
answerContent →  text image? | image
image         →  'IMAGE:' (STRING_LITERAL | text)
```

### Lexer Rules (tokens)

| Token | Pattern | Example |
|---|---|---|
| `OPTION_LABEL` | `A`–`E` | `A`, `B`, `C` |
| `CORRECT_MARKER` | `*` | marks the correct answer |
| `BOOLEAN` | `true` / `false` | metadata values |
| `IDENTIFIER` | letter then `[a-zA-Z0-9_]*` | `TIME_LIMIT`, `font` |
| `NUMBER` | digits, optional decimal and `%` | `60`, `5`, `14` |
| `STRING_LITERAL` | `"..."` | `"assets/eiffel.jpg"` |
| `TEXT_CONTENT` | alphanumeric + spaces/punctuation | `What is 2 plus 2?` |
| `WS` | whitespace | skipped |

> **Note:** Because `TEXT_CONTENT` greedily consumes spaces, some formatting rules apply to `.qm` files — see the [Input Format](#input-format) section below.

## Data Model (`models.py`)

Four dataclasses represent the parsed quiz in memory:

```python
QuizData
├── title: str
├── theme: ThemeData | None
│   ├── properties: dict        # e.g. {"primary_color": "blue"}
│   └── styles: dict            # e.g. {"question": {"font": "Verdana", "size": 14}}
├── metadata: dict              # e.g. {"TIME_LIMIT": 60, "DIFFICULTY": "Easy"}
└── questions: list[QuestionData]
    ├── text: str
    ├── points: int | float | None
    ├── image: str | None
    └── answers: list[AnswerData]
        ├── label: str          # "A", "B", "C" …
        ├── text: str | None
        ├── image: str | None
        └── correct: bool
```

## The Visitor (`QuizVisitorImpl.py`)

`QuizVisitorImpl` extends the generated `QuizVisitor` base class and overrides every `visit*` method. Each method maps 1-to-1 with a grammar rule and returns a typed value:

| Method | Grammar rule | Returns |
|---|---|---|
| `visitQuiz` | `quiz` | `QuizData` |
| `visitThemeBlock` | `themeBlock` | `ThemeData` |
| `visitThemeStatement` | `themeStatement` | mutates `ThemeData` |
| `visitSelector` | `selector` | `str` |
| `visitAttributeList` | `attributeList` | `dict` |
| `visitAttribute` | `attribute` | `(key, value)` tuple |
| `visitMetadata` | `metadata` | `(key, value)` tuple |
| `visitQuestion` | `question` | `QuestionData` |
| `visitPoints` | `points` | `int` or `float` |
| `visitQuestionMedia` | `questionMedia` | `str` (image path) |
| `visitAnswer` | `answer` | `AnswerData` |
| `visitAnswerContent` | `answerContent` | `(text, image)` tuple |
| `visitImage` | `image` | `str` (path, quotes stripped) |
| `visitValue` | `value` | `str`, `int/float`, or `bool` |
| `visitText` | `text` | `str` |

This differs from the alternative **Listener** pattern (also generated). A listener uses event callbacks (`enterQuiz`, `exitQuiz`, …) fired automatically as ANTLR walks the tree — useful for side-effects like building a symbol table. The visitor pattern used here is better suited for building a return value (the data model) because each `visit*` call explicitly returns data up the call stack.

## Entry Point (`main.py`)

`main.py` only does three things:

1. **Lex + Parse** — feeds the file through `QuizLexer` and `QuizParser`
2. **Visit** — calls `QuizVisitorImpl().visitQuiz(tree)` to get a `QuizData` object
3. **Print** — passes the `QuizData` to `print_quiz()` for display

## Input Format

A `.qm` file must follow these conventions:

- **Quiz title** — any text/identifier after `QUIZ:`
- **Theme block** (optional) — `THEME` keyword, then `{` on the same or next line; no space between a selector keyword and `[`
- **Metadata** (optional, repeatable) — `KEY: value` lines between the theme block and first question
- **Questions** — `QUESTION:` or `QUESTION(points=N):` (no space before `(`)
- **Answers** — `A:` through `E:`, with `*` after the content to mark correct
- **Images** — `IMAGE: "path/to/file"` quoted paths, either on the question line or as the answer content
- **Attribute values** inside `[...]` must be quoted strings (`"Verdana"`) or bare numbers

### Example

```
QUIZ: Science 101
THEME
{
    primary_color = "blue"
    question[font="Verdana", size=14]
    correct[color="green", weight="bold"]
}
TIME_LIMIT: 60
DIFFICULTY: Easy
CATEGORY: Biology

QUESTION: Which of these is a mammal?
A: Shark
B: Whale *
C: Lizard
```

### Example with image

```
QUIZ: World Landmarks
QUESTION(points=5): Name this structure
IMAGE: "assets/eiffel.jpg"
A: Eiffel Tower *
B: Statue of Liberty
C: Colosseum
```
